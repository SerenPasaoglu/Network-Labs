## Two-City Enterprise Network

## Objective
The purpose of this lab is to design a small enterprise network connecting two different cities while implementing routing, switching, DHCP, DNS, and web services.

## Topology
- 2 router
- 2 switch
- 2 server
- 5 clients
- 2 printers

## Configuration & Testing

- Two regions were defined: Izmir and Istanbul.
- A laptop, printer, and server were connected via the switch.
- Client IPs were automatically obtained by configuring them through the DHCP server, not manually.
- Configured router interfaces with IP addresses.
- Implemented static routing between the two routers.
- Verified communication between the Izmir and Istanbul networks.
- A DNS connection was established from the Istanbul server, and a web search was performed from the Izmir client.

The connection between each client, switch, and router was tested, and the transmission was observed to be successful and accurate.

## ResulT

This lab successfully simulated a small enterprise network connecting two remote locations. 
I gained practical experience in configuring routers, switches, DHCP, DNS, and web services within the same network environment. 
I also learned how static routing enables communication between different networks and how multiple network services work together to provide end-to-end connectivity.
